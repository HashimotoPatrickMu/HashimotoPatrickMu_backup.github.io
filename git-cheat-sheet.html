Git Command Summary

Configuring Git Settings

Set configuration options for the local repository:

git config user.name "Your Name"
git config user.email "your.email@example.com"

Check configuration settings:

git config --list
git config user.name
git config user.email

Pushing Changes

Push changes to the remote repository:

git push origin <branch-name>

Set upstream branch and push:

git push -u origin <branch-name>

Undoing Commits

Soft reset (undo commit but keep changes):

git reset --soft HEAD~1

Hard reset (undo commit and discard changes):

git reset --hard HEAD~1

Force push after reset (use with caution):

git push origin <branch-name> --force

Merging Branches

Switch to the branch you want to merge into:

git checkout main

Merge another branch into your current branch:

git merge <branch-to-merge>

Synchronizing with Remote Repository

Using git pull to Fetch and Merge

git pull combines fetching from the remote and merging into the current branch:

git pull origin <branch-name>

Using git fetch and git merge

Fetch updates from remote without automatic merge, then manually merge:

git fetch origin
git merge origin/<branch-name>

Differences between git pull and git fetch + git merge:

  • git pull: Automatically fetches and merges changes from the remote branch to your current branch.
  • git fetch + git merge: Fetches changes from all branches without merging. Allows you to review changes before manually merging them into a specific branch.
